# Glossary

## ACF — Advanced Custom Fields
A WordPress plugin that allows you to add **structured fields** (text boxes, toggles, links, numbers, etc.) to content.  
ACF is used to keep content separate from design so editors fill in fields instead of editing layouts.

---

## API — Application Programming Interface
A way for one system to talk to another in a structured way.  
In this project, APIs may later be used to pull data (e.g., statistics or topics) from external systems like Discourse.

---

## Archive (Template)
A WordPress page type that displays a **list of content items**, such as all posts in a category or all entries in a Custom Post Type.

---

## A11y — Accessibility
A shorthand for **accessibility** (the “11” represents the letters between “a” and “y”).  
Refers to making the site usable by people with disabilities (screen readers, keyboard navigation, contrast, etc.).

---

## CDN — Content Delivery Network
A distributed network that delivers site assets (images, CSS, JS) from servers close to the user to improve performance and reliability.

---

## CMS — Content Management System
Software that allows non-technical users to manage content.  
WordPress is the CMS for this project.

---

## CPT — Custom Post Type
A WordPress feature that lets you create **new kinds of content** beyond posts and pages.  
Examples: Hot Topics, Resources, Events.

---

## CTA — Call To Action
A prompt that tells the user what to do next.  
Examples: “Join the Forum,” “Read More,” “Get Started.”

Primary CTA = main action  
Secondary CTA = alternative or lower-priority action

---

## DB — Database
The system where WordPress stores content, settings, users, and configuration.  
Typically MySQL or MariaDB.

---

## Dev — Development Environment
A non-public environment used for building and experimenting.  
Often local (on a laptop) or on a private server.

---

## DNS — Domain Name System
The system that translates human-readable domain names (like `tiaa-forum.org`) into IP addresses.

---

## Elementor
A WordPress page builder plugin used as the **presentation layer** for this site.  
It controls layout, styling, and templates.

---

## Export (Elementor / ACF)
A file (usually JSON) created to move templates or field definitions between environments or to store them in Git.

---

## Git
A version control system used to track changes in code, documentation, and configuration files.

---

## GitHub
A hosted service for Git repositories.  
Used to share and manage the `tiaa-wpsite-v3` repository.

---

## Hot Topics
A project-specific Custom Post Type representing curated discussion topics (often linked to forum threads).

---

## IDE — Integrated Development Environment
A software tool used for writing and managing code.  
Example: IntelliJ IDEA.

---

## JSON — JavaScript Object Notation
A structured text format used for data exchange.  
Elementor templates and ACF field groups are often exported as JSON files.

---

## Loop (Elementor / WordPress)
A mechanism that automatically displays multiple items (posts, CPT entries) using the same layout template.

---

## MU Plugin — Must-Use Plugin
A WordPress plugin that loads automatically and **cannot be disabled via the admin UI**.  
Used for critical site logic.

---
## MVP -/ Minimum Viable Product
A simplified version of the site that can be used to demonstrate core functionality.

---

## Plugin (WordPress)
A package of code that extends WordPress functionality.  
Plugins can usually be enabled/disabled from the admin interface.

---

## Prod — Production Environment
The live, public version of the site that real users access.

---

## Repo — Repository
A directory tracked by Git that contains code, documentation, and related assets.

---

## SEO — Search Engine Optimization
Practices that help search engines understand and rank site content.

---

## Staging / Test
A public or semi-public environment used to test changes before they go to production.

---

## Template (Elementor / WordPress)
A reusable layout definition that controls how content is displayed.  
Examples: Header, Footer, Single Post, Archive.

---

## Theme (WordPress)
A package that controls a site’s overall presentation.  
This project uses the lightweight **Hello** theme with Elementor.

---

## Theme Builder (Elementor)
An Elementor feature that allows building site-wide templates (headers, footers, single pages, archives) without writing PHP.

---

## UI — User Interface
The visual elements a user interacts with (buttons, menus, forms).

---

## URL — Uniform Resource Locator
The address of a resource on the web (e.g., `https://example.org/page`).

---

## UX — User Experience
How easy, intuitive, and pleasant the site is to use.

---

## WP — WordPress
The CMS platform used for this site.

---

## WP-CLI — WordPress Command Line Interface
A tool for managing WordPress sites via the command line (used for automation and maintenance).

---

## YAML — YAML Ain’t Markup Language

Sometimes expressed as **YML** (the normal file extension) — A human-readable data format sometimes used for configuration files (not heavily used here, but commonly referenced).
